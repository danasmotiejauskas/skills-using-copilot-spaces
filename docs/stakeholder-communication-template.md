Use this as the canonical weekly/milestone update to simplify status and reduce ad-hoc questions.

- Project: [name]
- Update period / Date:
- Prepared by: [PM / PdM]
- Summary (1–2 sentences):
- Progress this period:
  - Completed:
  - In progress:
- Key metrics / success signals:
- Risks & blockers:
  - [ID] Short description — Impact — Owner — Mitigation
- Decisions needed:
  - [Decision], due by [date], owner [Stakeholder or role]
- Upcoming milestones:
  - [milestone] — target date
- Asks / support required from stakeholders:
- Attachments / links:
  - Project One-pager:
  - Roadmap:
  - Risk register:
  - Demo recording / design files:

Guidance:
- Keep it concise (target 4–8 bullets per section).
- Use the same template each period to make trends easier to follow.
